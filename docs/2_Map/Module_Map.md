## 二、节点类型（Demo 4种）

| 节点 | 图标 | Demo 实现 |
|------|------|-----------|
| 普通战斗 | ⚔️ | 标准战棋战斗，1波敌人 |
| 商店 | 🛒 | 消耗金币购买消耗品/装备 |
| 篝火 | 🔥 | 恢复30%HP + 驱散1个负面状态 |
| 事件 | ❓ | 2-3个预设事件模板 |

> [Demo] 不做精英战斗、宝箱、谜题棋盘。4种节点足够展示肉鸽路线选择。

## 三、楼层生成参数

```json
{
  "floor": 3,
  "grid_size": [10, 10],
  "terrain": "stone_dungeon",
  "enemy_count": 3,
  "enemy_types": ["slime", "demon_soldier"],
  "obstacles": 4
}
```

| 参数 | 1-3层 | 4-5层 | 6-9层 | 10层 |
|------|-------|-------|-------|------|
| 网格 | 10×10 | 12×12 | 14×14 | 16×16 |
| 敌人数量 | 1-3 | 3-4 | 4-5 | Boss×1 |
| 障碍物 | 2 | 4 | 5 | 6 |

---

## 四、地图生成算法（简化）

```
1. 创建 N×N 正方形网格
2. 随机放置障碍物（石柱），避开部署区（边缘2格）
3. 随机放置友方部署位（底边2格内）
4. 随机放置敌人部署位（顶边2格内）
5. 验证：每个格子至少有一条路径可达
```

> [Demo] 不实现高度差、水域、熔岩等地形变体。全部使用平地 + 障碍物。
